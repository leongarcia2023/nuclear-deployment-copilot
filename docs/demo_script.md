# Nuclear Deployment Intelligence Demo Script

## 60-second product explanation

Nuclear Deployment Intelligence turns advanced nuclear deployment claims into source-grounded diligence memos. A user pastes a public claim or sanitized note, selects their role and decision question, and the system decomposes the claim into deployment evidence requirements.

This version is deterministic and does not use paid AI API calls. It separates public corpus context from target-specific support, flags missing or private evidence, and produces a concise first-pass IC memo plus diligence questions.

## 3-minute demo script

1. Open the Deal Diligence page.
2. Click **Data center power campus**.
3. Click **Generate first-pass diligence memo**.
4. Point to the verdict, confidence, one-line judgment, and analyst read.
5. Show the top missing evidence and evidence coverage summary.
6. Show the relevant public evidence bullets and explain that public precedent is not treated as target-specific proof.
7. Copy the memo and note that it is designed for an IC prep note or counterparty-call brief.
8. Open Source Library briefly to show the curated public-source foundation.
9. Open Methodology briefly to show the deployment risk framework.

## 5-minute demo script

1. Start with the 60-second explanation.
2. Run **Data center power campus** and explain why the first diligence question is power-campus deliverability, not reactor design.
3. Run **HALEU fuel-readiness claim** and show how public HALEU context does not prove target-specific fuel supply.
4. Run **DOE award vs financing** and show how the memo distinguishes strategic funding signals from closed project finance.
5. Open Source Library and filter or search for a relevant source family.
6. Open Methodology and emphasize the deployment layers: licensing, fuel, site, interconnection, EPC, offtake, financing, and operations.

## Recommended demo preset order

1. Data center power campus
2. HALEU fuel-readiness claim
3. NRC pre-application to 2030
4. DOE award vs financing
5. MOU vs binding offtake

## Key talking points

- The product is claim-to-memo, not a chatbot.
- It decomposes broad claims into evidence requirements.
- It separates corpus coverage from target-specific support.
- It treats private contracts, financing terms, site rights, and confidential counterparties as private diligence items.
- It is intentionally deterministic for the MVP so the demo is cheap to run and repeatable.

## Limitations to mention honestly

- This version does not verify confidential documents or private contracts.
- It does not yet run AI synthesis or vector retrieval.
- It uses a curated seed corpus rather than a complete nuclear document universe.
- It supports first-pass diligence, not final investment judgment.

## Roadmap

- V2: Expand the curated public-source corpus and ingestion coverage.
- Optional later AI synthesis: Add cited synthesis once the evidence framework is stable.
- V3: Add siting intelligence for location, interconnection, water, permitting, and local risk.
